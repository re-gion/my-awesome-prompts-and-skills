# Style Checklist

Use this checklist during the final polish pass.

## Human, Pragmatic Chinese Voice

- Write like a capable teacher, not a product brochure.
- Prefer short direct sentences for instructions.
- Use longer sentences only when connecting ideas genuinely helps.
- Keep a little personality, but do not force jokes or drama.
- Say "这里容易踩坑" when there is a real pitfall; do not manufacture tension.

## Remove AI-Sounding Patterns

- Delete filler openings such as "在当今快速发展的技术时代".
- Avoid exaggerated phrases such as "至关重要", "深刻变革", "不可或缺", "全面赋能" unless evidence demands them.
- Avoid vague attribution: replace "有观点认为" with a named source, or remove it.
- Avoid forced three-part lists when two items are enough.
- Avoid repeated "不仅...而且..." structures.
- Avoid quote-like closing slogans.

## Explain Concepts

- Use analogies for hard abstractions only: sandbox, virtual machine, shell, package manager, prompt context, agent loop, permissions, network proxy, etc.
- Keep each analogy short and return to the technical reality immediately.
- For ordinary tips, use concise instructions instead of metaphors.

## Tutorial Rhythm

- Start from a working baseline before advanced details.
- Put commands near the explanation they support.
- After a dense section, add a quick check or expected result.
- Put troubleshooting after the main path, not before the learner has tried anything.

## Final Read-Through

- Check that headings form a learning path from easy to hard.
- Check that every command has a purpose and expected outcome.
- Check that current-version claims have dates, versions, or source notes.
- Remove anything that sounds polished but does not help the learner act.
